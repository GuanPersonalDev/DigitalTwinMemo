# 數位孿生

Digital Twin

## 定義

> 數位孿生是實體物件、流程、服務或系統的數位副本或虛擬表示。它透過雙向資料流持續更新物理對應物，是一種迭代式的自主系統建構方法。

— Dihan et al., 2024

## 詳細解釋

數位孿生不只是 3D 模型或靜態副本，而是：
- **動態的**：持續與物理世界同步
- **智慧的**：整合 AI/ML 進行預測和優化
- **雙向的**：虛擬變更可反饋到物理世界

### 歷史

- **1960s**：NASA 阿波羅任務首次使用概念
- **2010s**：工業 4.0 推動製造業採用
- **2020s**：擴展至城市、醫療、農業等領域

## 核心組成

### 三大元素

1. **物理實體（Physical Entity）**
   - 感測器、致動器、機器、設備
   - 產生即時資料

2. **虛擬模型（Virtual Model）**
   - 幾何模型、物理模型、行為模型
   - 模擬與預測

3. **資料連接（Data Connection）**
   - 雙向資料流
   - IoT、API、通訊協定

### 資料處理流程

```
收集 → 儲存 → 關聯 → 融合 → 排序 → 協調
```

## 應用場景

| 領域 | 應用 | 關鍵技術 |
|------|------|----------|
| **製造業** | 工廠規劃、預測性維護 | OPC-UA, MQTT, Big Data |
| **城市化** | 智慧城市、交通管理 | BIM, LiDAR, 群眾外包 |
| **農業** | 作物管理、資源優化 | IoT, LoRa, 無人機 |
| **醫療** | 個人化醫療、手術模擬 | 穿戴裝置, 深度學習 |
| **機器人** | 訓練、模擬、協作 | ROS, Unity 3D, VR/AR |
| **航空** | 飛行模擬、維護預測 | 特徵模擬技術 |

## 技術架構

### 典型三層架構

1. **Physical Layer**：感測器、設備、資料收集
2. **Edge Layer**：邊緣運算、即時推論
3. **Cloud Layer**：中央協調、長期分析
