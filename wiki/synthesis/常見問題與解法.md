# 常見問題與解法

開發數位孿生專案時遇到的問題與解決方案。

## Docker 相關

### docker compose up -d 找不到設定檔

**錯誤訊息**：找不到 docker-compose.yml

**原因**：PowerShell 不在專案目錄

**解法**：
```bash
cd /path/to/project  # 切換到專案根目錄
docker-compose up -d
```

---

### Port 1883 already allocated

**錯誤訊息**：`Bind for 0.0.0.0:1883 failed: port is already allocated`

**原因**：之前的 Mosquitto 容器還在運行

**解法**：
```bash
# 查看佔用端口的容器
docker ps

# 停止舊容器
docker stop <container_name>

# 或強制移除
docker rm -f <container_name>
```

---

### factory_mosquitto 一直 Restarting

**錯誤訊息**：容器狀態顯示 `Restarting`

**原因**：`mosquitto.conf` 設定檔有拼寫錯誤

**常見拼錯**：
```diff
- persistance true      # ❌ 錯誤拼寫
+ persistence true      # ✓ 正確
```

**解法**：
```bash
# 檢查日誌找出錯誤
docker logs factory_mosquitto

# 修正設定檔後重啟
docker-compose restart
```

## WSL2 網路相關

### ConnectionRefusedError（localhost 連不到 Docker）

**錯誤訊息**：`ConnectionRefusedError: [Errno 111] Connection refused`

**原因**：WSL2 使用 `localhost` 無法連到 Windows 上的 Docker

**解法**：改用 Windows 實際 IP

```bash
# 在 WSL2 中取得 Windows IP
cat /etc/resolv.conf | grep nameserver
# 輸出：nameserver 172.x.x.x

# 使用該 IP 連接
MQTT_BROKER_HOST = "172.x.x.x"  # 改用 Windows IP
```

---

### TimeoutError（host.docker.internal 無法解析）

**錯誤訊息**：連線逾時

**原因**：`host.docker.internal` 在 WSL2 環境中無法解析

**解法**：改用實際 Windows IP（同上）

```python
# config.py
# MQTT_BROKER_HOST = "host.docker.internal"  # ❌ WSL2 不支援
MQTT_BROKER_HOST = "172.x.x.x"               # ✓ 使用實際 IP
```

**取得 IP 的方式**：
```bash
# 方法一：從 resolv.conf
cat /etc/resolv.conf | grep nameserver | awk '{print $2}'

# 方法二：PowerShell（Windows 端）
ipconfig | findstr "IPv4"
```

## Python / paho-mqtt 相關

### ModuleNotFoundError: paho

**錯誤訊息**：`ModuleNotFoundError: No module named 'paho'`

**原因**：WSL2 環境沒有安裝 paho-mqtt

**解法**：
```bash
pip install paho-mqtt
```

---

### mqtt.Client() DeprecationWarning

**錯誤訊息**：`DeprecationWarning: Callback API version 1 is deprecated`

**原因**：paho-mqtt 2.0+ 版本 API 異動

**解法**：
```python
# 舊寫法（已棄用）
client = mqtt.Client()

# 新寫法
client = mqtt.Client(mqtt.CallbackAPIVersion.VERSION2)
```

---

### from paho.mqtt.client import mqtt 報錯

**錯誤訊息**：`ImportError: cannot import name 'mqtt'`

**原因**：import 方式錯誤

**解法**：
```python
# ❌ 錯誤
from paho.mqtt.client import mqtt

# ✓ 正確
import paho.mqtt.client as mqtt
```

## Omniverse Extension 相關

### Extension Manager 找不到 Extension

**現象**：在 Extension Manager 中搜尋不到自己開發的 Extension

**原因**：`kit` 檔的 `folders.'++'` 沒有加入 `source/extensions` 路徑

**解法**：

編輯 `.kit` 設定檔，加入 Extension 搜尋路徑：

```toml
[settings.app.exts]
folders.'++'= ["${app}/../../../../source/extensions"]
```

**說明**：
- `${app}` 代表 Omniverse 應用程式根目錄
- `../../../../` 回溯到專案目錄
- `source/extensions` 為 Extension 存放位置

---

### No module named 'paho'

**錯誤訊息**：`ModuleNotFoundError: No module named 'paho'`

**原因**：Omniverse 使用自己內建的 Python 環境，與系統 Python 獨立

**解法**：使用 Omniverse 內部的 `python.exe` 安裝套件

```powershell
# 找到 Omniverse 的 Python（路徑依版本而異）
# 例如 kit-app-template 專案
cd C:\path\to\kit-app-template\_build\windows-x86_64\release

# 使用該 Python 安裝 paho-mqtt
.\python.exe -m pip install paho-mqtt
```

**注意**：
- 不同 Omniverse 應用程式可能有各自的 Python 環境
- 每次建置後可能需要重新安裝
- 可考慮在 `extension.toml` 中加入依賴設定

---

### MQTT 連線失敗（WinError 10061）

**錯誤訊息**：`[WinError 10061] 無法連線，因為目標電腦主動拒絕`

**原因**：Docker Mosquitto 服務未啟動

**解法**：

```powershell
# 切換到專案目錄
cd C:\path\to\factory-floor-digital-twin

# 啟動 Docker 服務
docker compose up -d

# 確認 Mosquitto 運行中
docker ps
```

**檢查清單**：
1. Docker Desktop 是否已啟動？
2. `docker-compose.yml` 是否在當前目錄？
3. Port 1883 是否已被佔用？

---

### Extension 與 Git 專案管理衝突

**問題**：`kit-app-template` 的 extensions 資料夾與 `factory-floor-digital-twin` 專案分開管理，難以同步

**解法**：使用 Windows Junction（符號連結）

```powershell
# 在 kit-app-template/source/extensions 建立 Junction
# 連結到 factory-floor-digital-twin/omniverse_extension

mklink /J "C:\path\to\kit-app-template\source\extensions\omniverse_factory_twin" ^
          "C:\path\to\factory-floor-digital-twin\omniverse_extension\omniverse_factory_twin"
```

**優點**：
- Extension 原始碼保留在 `factory-floor-digital-twin` 專案中
- Git 追蹤不受影響
- Omniverse 可以正常載入 Extension
- 修改會即時同步

**注意**：
- Junction 是 Windows 特有功能（類似 Linux 的 symlink）
- 刪除 Junction 不會刪除原始資料夾
- 需要以管理員權限執行 `mklink /J`

### ChangeProperty 執行失敗

**錯誤訊息**：`omni.kit.commands.execute` 執行失敗或無效

**原因**：`omni.kit.commands.execute` 不能在 MQTT 背景執行緒直接呼叫，USD 操作必須在主執行緒中執行

**解法**：改用 USD API 直接操作

```python
# ❌ 錯誤：使用 commands（可能有執行緒限制）
omni.kit.commands.execute("ChangeProperty", ...)

# ✓ 正確：直接使用 USD API
UsdGeom.Gprim(prim).GetDisplayColorAttr().Set([(r, g, b)])
```

**關鍵**：所有 USD 場景操作都應在主執行緒中執行，使用 `create_subscription_to_pop()` 確保執行緒安全。

---

### Cube 顏色沒有即時更新

**現象**：MQTT 訊息收到了，但 3D 物件顏色沒有變化

**原因**：`get_update_event_stream().push()` 在某些 Kit 版本無效

**解法**：改用 `create_subscription_to_pop()` 訂閱每幀更新事件

```python
# ❌ 無效：push 事件
omni.kit.app.get_app().get_update_event_stream().push()

# ✓ 正確：訂閱更新事件，每幀處理
self.updateSub_ = omni.kit.app.get_app().get_update_event_stream()\
    .create_subscription_to_pop(
        self.onUpdate,
        name="factory_twin_update"
    )

def onUpdate(self, event):
    # 從 pendingUpdates_ 取出待更新項目
    with self.lock_:
        updates = dict(self.pendingUpdates_)
        self.pendingUpdates_.clear()
    for path, color in updates.items():
        self.updateColor(path, color)
```

### lock_ 找不到（AttributeError）

**錯誤訊息**：`AttributeError: 'FactoryTwinExtension' object has no attribute 'lock_'`

**原因**：`onExtensionStartup` 的簽名缺少 `ext_id` 參數，導致子類別覆寫失敗，`lock_` 從未初始化

**錯誤程式碼**：
```python
# base_extension.py
def onExtensionStartup(self, ext_id):  # 有 ext_id
    pass

# extension.py（子類別）
def onExtensionStartup(self):  # ❌ 缺少 ext_id，覆寫失敗
    self.lock_ = threading.Lock()
```

**解法**：統一方法簽名

```python
# extension.py（子類別）
def onExtensionStartup(self, ext_id):  # ✓ 加上 ext_id
    self.lock_ = threading.Lock()
    self.pendingUpdates_ = {}
```

**教訓**：覆寫方法時，簽名必須完全匹配父類別，否則會建立新方法而非覆寫。

### 重啟後重複 subscribe

**現象**：Extension 重啟後，收到重複的 MQTT 訊息

**原因**：`on_shutdown` 未定義，Extension 重啟時舊的 MQTT 連線沒有釋放

**解法**：在 `base_extension.py` 中實作 `on_shutdown()`

```python
# base_extension.py
def on_shutdown(self):
    print(f"[{self.__class__.__name__}] shutdown")
    self.onExtensionShutdown()  # 子類別清理 Hook
    if hasattr(self, 'mqttClient_') and self.mqttClient_:
        self.mqttClient_.disconnect()
        self.mqttClient_ = None
```

**檢查清單**：
1. `on_shutdown()` 是否有定義？
2. MQTT 客戶端是否有呼叫 `disconnect()`？
3. 背景執行緒是否有呼叫 `loop_stop()`？

## 拼寫錯誤（Typo）

### Topic 收不到資料

**現象**：Publisher 有發送，但 Subscriber 收不到

**原因**：`config.py` 中 topic 名稱拼寫錯誤

**常見拼錯**：
```python
# ❌ 錯誤
"/factory/machine_01/statue"   # statue（雕像）

# ✓ 正確
"/factory/machine_01/status"   # status（狀態）
```

**除錯方式**：
```bash
# 列出所有 ROS2 topic
ros2 topic list

# 監聽特定 topic 確認資料
ros2 topic echo /factory/machine_01/status
```

---

### mosquitto.conf 設定無效

**常見拼錯**：
```diff
- persistance true      # ❌
+ persistence true      # ✓

- allow_annonymous true # ❌
+ allow_anonymous true  # ✓
```

## 除錯檢查清單

遇到問題時，依序檢查：

1. **目錄位置**：是否在專案根目錄？
2. **容器狀態**：`docker ps` 確認服務運行中
3. **端口佔用**：`lsof -i :1883` 或 `netstat -an | grep 1883`
4. **網路連通**：`ping <ip>` 測試連線
5. **日誌檢查**：`docker logs <container>` 查看錯誤
6. **拼寫檢查**：topic 名稱、設定項是否正確
7. **套件版本**：API 是否因版本更新而改變
