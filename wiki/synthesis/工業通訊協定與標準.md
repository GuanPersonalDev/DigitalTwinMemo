# 工業通訊協定與標準

工業自動化與智慧製造中常用的通訊協定與國際標準。

## 通訊協定總覽

### 協定定位圖

```
                    雲端/企業層
                         ↑
              ┌──────────┴──────────┐
              │        MQTT         │  輕量、雲端通訊
              └──────────┬──────────┘
                         │
              ┌──────────┴──────────┐
              │       OPC UA        │  安全、語義豐富
              └──────────┬──────────┘
                         │
              ┌──────────┴──────────┐
              │   Modbus / Profinet │  現場設備通訊
              └──────────┬──────────┘
                         │
                    現場設備層
```

### 協定比較表

| 特性 | MQTT | OPC UA | Modbus |
|------|------|--------|--------|
| **定位** | 雲端/邊緣通訊 | 工廠內部 M2M | 現場設備 |
| **架構** | Pub/Sub | Client/Server | Master/Slave |
| **複雜度** | 低 | 高 | 低 |
| **安全性** | TLS（選配） | 內建完整安全 | 無 |
| **語義模型** | 無 | 豐富資訊模型 | 無 |
| **頻寬需求** | 極低 | 中等 | 低 |
| **適用場景** | IoT 感測器、雲端 | 工廠自動化 | 舊設備整合 |

## MQTT

**你的專案已使用**

### 基本資訊

- **全名**：Message Queuing Telemetry Transport
- **架構**：發布/訂閱（Pub/Sub）
- **預設端口**：1883（明文）、8883（TLS）

### 優點

- 極輕量（最小封包僅 2 bytes）
- 低頻寬、低功耗
- 容易實作
- 適合不穩定網路

### 缺點

- 無內建語義模型
- 安全性需額外配置
- 不適合複雜資料結構

### 適用場景

```
感測器 → Edge Gateway → MQTT Broker → 雲端平台
                              ↑
                        你的專案位置
```

## OPC UA

### 基本資訊

- **全名**：Open Platform Communications Unified Architecture
- **架構**：Client/Server
- **標準**：IEC 62541

### 核心特點

1. **資訊模型**：定義資料結構與語義
2. **安全性**：認證、授權、加密
3. **跨平台**：Windows、Linux、嵌入式
4. **向下相容**：可橋接 OPC Classic

### 資訊模型範例

```
Machine_01 (Object)
├── Temperature (Variable)
│   ├── Value: 75.5
│   ├── Unit: "Celsius"
│   └── Timestamp: 2026-04-18T15:30:00Z
├── Status (Variable)
│   └── Value: "Running"
└── Methods
    ├── Start()
    └── Stop()
```

### 適用場景

- 工廠內部 M2M 通訊
- MES ↔ PLC 整合
- 需要豐富語義的場景

## Modbus

### 基本資訊

- **發明**：1979 年（Modicon）
- **架構**：Master/Slave
- **變體**：Modbus RTU、Modbus TCP

### 暫存器類型

| 類型 | 功能碼 | 讀/寫 | 說明 |
|------|--------|-------|------|
| Coils | 01, 05, 15 | R/W | 數位輸出 |
| Discrete Inputs | 02 | R | 數位輸入 |
| Holding Registers | 03, 06, 16 | R/W | 類比輸出 |
| Input Registers | 04 | R | 類比輸入 |

### 適用場景

- 舊設備整合
- PLC ↔ SCADA
- 簡單感測器讀取

## 2025 主流架構

**OPC UA + MQTT 混合架構**

```
┌─────────────────────────────────────────────┐
│                  雲端平台                    │
│            (Azure IoT / AWS IoT)            │
└──────────────────┬──────────────────────────┘
                   │ MQTT
┌──────────────────┴──────────────────────────┐
│                Edge Gateway                  │
│         (協定轉換、資料聚合)                 │
└────┬─────────────┬─────────────┬────────────┘
     │ OPC UA      │ Modbus      │ MQTT
┌────┴────┐   ┌────┴────┐   ┌────┴────┐
│  PLC    │   │ 舊設備  │   │ 感測器 │
└─────────┘   └─────────┘   └─────────┘
```

**你的專案架構**：
```
ROS2 (現場) → MQTT (橋接) → 雲端/Omniverse
```

## 國際標準

### ISA-95 / IEC 62264（自動化金字塔）

企業與控制系統整合標準，定義五層架構：

```
Level 4 ─────────────────────────────────────
│  企業層（ERP）                              │
│  訂單、庫存、財務、物流                     │
├─────────────────────────────────────────────
Level 3 ─────────────────────────────────────
│  製造執行層（MES）                          │
│  生產排程、品質管理、設備維護               │
│  ⬆ IT/OT 邊界 ⬆                            │
├─────────────────────────────────────────────
Level 2 ─────────────────────────────────────
│  監控層（SCADA/HMI）                        │
│  製程監控、順序控制                         │
├─────────────────────────────────────────────
Level 1 ─────────────────────────────────────
│  控制層（PLC/DCS）                          │
│  設備控制、迴路控制                         │
├─────────────────────────────────────────────
Level 0 ─────────────────────────────────────
│  現場層                                     │
│  感測器、執行器、機台                       │
└─────────────────────────────────────────────
```

**你的專案對應**：
| 層級 | 元件 |
|------|------|
| Level 0 | 模擬感測器（temperature, vibration） |
| Level 1-2 | ROS2 Publisher |
| Level 2-3 | MQTT Bridge |
| Level 3+ | Omniverse 視覺化 |

---

### ISA-88 / IEC 61512（批次控制）

批次製造的程序控制標準：

- **物理模型**：Enterprise → Site → Area → Cell → Unit
- **程序模型**：Procedure → Unit Procedure → Operation → Phase
- **適用**：製藥、化工、食品

---

### IEC 62443（工業資安）

工業自動化控制系統（IACS）的資安標準。

#### 標準結構

| 系列 | 名稱 | 內容 |
|------|------|------|
| 62443-1 | General | 術語、概念、模型 |
| 62443-2 | Policies | 資安管理系統 |
| 62443-3 | System | 系統層級安全需求 |
| 62443-4 | Component | 元件開發與功能需求 |

#### 安全等級（SL）

| 等級 | 威脅來源 | 說明 |
|------|---------|------|
| SL 1 | 意外誤用 | 基本保護 |
| SL 2 | 一般攻擊者 | 低資源攻擊 |
| SL 3 | 熟練攻擊者 | 中等資源 |
| SL 4 | 國家級攻擊 | 高度資源 |

#### 區域與導管（Zones & Conduits）

```
┌─────────────────┐    Conduit    ┌─────────────────┐
│    Zone A       │◄────────────► │    Zone B       │
│  (Control)      │   (防火牆)    │  (Enterprise)   │
│   SL 3          │               │   SL 2          │
└─────────────────┘               └─────────────────┘
```

## 協定選擇指南

### 依場景選擇

| 場景 | 建議協定 |
|------|---------|
| 感測器到雲端 | MQTT |
| PLC 到 MES | OPC UA |
| 舊設備整合 | Modbus → Gateway |
| 機器人通訊 | ROS2 / DDS |
| 即時控制 | EtherCAT / Profinet |

### 你的專案建議

```
目前架構（OK）：
ROS2 → MQTT → Cloud/Omniverse

未來可考慮：
OPC UA Server（提供標準化介面給其他系統）
```
